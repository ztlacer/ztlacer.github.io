RentalManager

Using data structures I had built all up from scratch, I created an application in my Data Structures course that would take in information on rental hosts, with costs associated with a timeframe they would be open for, and translate them into a graph struture.

Then, using the days these rentals were open as nodes and the rent cost between days as the edge weight, I coded a solution for finding the cheapest string of rentals that would match the timeframe a user would want to consecutively rent out a space to live in. Afterwards, I analyzed the runtime of this solution using different types of graphs for the overall graph structure.



WordCloud

Using the Queue, Stack, and Map data structures, which I had implemented from scratch, I developed an application that would take in texts and list their most common words. The user can also specifiy which words they want to see the frequency of, as well as add in a filter text which specifies which words in the original text to be ignored. Using exponentially larger texts, I experiemented with and analyzed the asymptotic analysis of the program's runtime by using different Map types along with different text sizes.

Cracking the Code Midway

This was my first Unity game made early in my college career. The game is based on WWII and the battle of Midway. Here the player takes on the role of an American soldier who's task is to decrypt intercepted messages from the Japanese Navy. The game play is based on how this happened in history, with the Japanese encrypting their words into Romanji (english spellings of Japanese words) before transmission and using code books to recompose the message. Using each level's code book, the player decyphers each message by using context clues and pattern recognition.
